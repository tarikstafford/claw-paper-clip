   - Define a `ChatMessage` interface: `{ id: string; threadId: string; senderType: string; body: string; createdAt: string }`

   - `**bold**` to `<b>bold</b>`
   - `*italic*` to `<i>italic</i>` (must run AFTER bold replacement)
   - `` `code` `` to `<code>code</code>`
   - `# Heading` (any level) to `<b>Heading</b>`
   - Remove `---` horizontal rules
   - Collapse 3+ newlines to 2
   - Trim
3. Keep the fallback logic: if Telegram returns "can't parse entities", retry without parse_mode (plain text).
4. All existing exports (`sendMessage`, `setWebhook`, `TOKEN`, `TelegramUpdate`, `TelegramEntity`) remain unchanged.

**conversation-manager.ts** — Full rewrite:
1. Import from new paperclip.ts: `createChatThread`, `postChatMessage`, `getNewMessages`, `CEO_AGENT_ID`.
2. Import `formatForTelegramHtml` from telegram.ts.
3. Import conversation-store functions (same names, new schema).

4. Rewrite `handleBoardMessage(chatId: number, username: string, text: string, telegramUpdateId: number): Promise<void>`:
   - Note the NEW 4th parameter `telegramUpdateId`.
   - Get conversation via `getConversation(chatId)`.
   - If no conversation or `conversation.threadId` is empty: call `createChatThread(chatId, username)`, upsert the conversation with the new threadId, then post the message.
   - If conversation exists with threadId: call `postChatMessage(conversation.threadId, text, telegramUpdateId)`.
   - If `postChatMessage` returns `'duplicate'`, log it and return early (no error to user).
   - On first message (new thread), also call `postChatMessage` after creating the thread to send the actual message content (the first message is NOT embedded in the thread title/description like the old issue approach).
   - On error, send the warning message to Telegram (same pattern as current code).

5. Rewrite `pollCeoReplies` as `pollAgentReplies(): Promise<void>`:
   - Iterate all conversations from `getAllConversations()`.
   - Skip entries with no threadId.
   - Call `getNewMessages(conversation.threadId, conversation.lastSeenMessageId)`.
   - Filter to `senderType === 'agent'` messages.
   - For each agent message, check against `forwardedMessageIds` Set (same dedup pattern), then call `sendMessage(conversation.chatId, formatForTelegramHtml(msg.body))`.
   - Update `lastSeenMessageId` to the latest message id (agent or not) via `updateLastSeen`.
   - Rename the Set from `forwardedCommentIds` to `forwardedMessageIds`.

6. Update `startPoller` to call `pollAgentReplies` instead of `pollCeoReplies`. Update log messages.
7. Export: `handleBoardMessage`, `pollAgentReplies`, `startPoller`.
