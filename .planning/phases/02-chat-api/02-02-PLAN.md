The test file must cover all 7 requirements. Structure as describe blocks:

**describe("POST /companies/:companyId/chat/threads"):**
- API-01: Creates thread with valid data, returns 201 with thread object containing companyId, agentId, creator info
- API-06: Board user sets creatorUserId; agent caller sets creatorAgentId
- Returns 401 for unauthenticated caller (req.actor.type === "none")

**describe("GET /companies/:companyId/chat/threads"):**
- API-02: Board user sees only threads where creatorUserId matches their userId
- API-02: Agent sees only threads where creatorAgentId matches their agentId
- API-06: Returns 401 for unauthenticated caller

**describe("GET /chat/threads/:threadId/messages"):**
- API-03: Returns messages in chronological order with nextCursor when results equal limit
- API-03: Returns nextCursor: null when fewer results than limit
- API-03: Cursor parameter (after=messageId) returns only messages after that message
- Returns 404 when thread does not exist
- Returns 403 when thread belongs to a different company than the actor

**describe("POST /chat/threads/:threadId/messages"):**
- API-04: User message is inserted with processingStatus="enqueued", returns 201
- API-05: User message triggers heartbeat.wakeup with correct agentId and contextSnapshot containing threadId and messageId
- API-07: Agent message sets senderType="agent", processingStatus="processed"
- API-07: Agent message does NOT trigger heartbeat.wakeup (critical — prevents infinite loop)
- API-06: Both board and agent callers can post messages
- Returns 404 when thread does not exist

**Mocking strategy:**
- Mock `chatService` functions to return predictable data (use vi.fn())
- Mock `heartbeatService(db).wakeup` to verify it's called/not called
- Mock `publishLiveEvent` to verify event publishing
- Mock `assertCompanyAccess` to allow or throw based on test scenario
- Mock `req.actor` to simulate board, agent, and none actor types

Each test should assert:
1. HTTP status code
2. Response body shape
3. Service function called with correct arguments
4. Side effects (wakeup called/not called, live event published)
