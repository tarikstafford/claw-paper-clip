Five endpoints — follow the exact patterns from 02-RESEARCH.md code examples:

**POST /companies/:companyId/chat/threads** (API-01):
- `validate(createThreadSchema)` middleware
- `assertCompanyAccess(req, companyId)`
- Check `req.actor.type !== "none"` -> throw unauthorized()
- Get actor info via `getActorInfo(req)`
- Create thread with creator columns set by actor type
- Return 201 with thread object

**GET /companies/:companyId/chat/threads** (API-02):
- `assertCompanyAccess(req, companyId)`
- Check `req.actor.type !== "none"` -> throw unauthorized()
- Filter: board users by userId, agents by agentId (per research recommendation and "list their threads" requirement)
- Return thread array

**GET /chat/threads/:threadId/messages** (API-03):
- Look up thread via `svc.getThread(threadId)`
- If not found, throw notFound("Thread not found")
- `assertCompanyAccess(req, thread.companyId)` — validates actor has access to thread's company
- Parse query params: `after` (cursor string), `limit` (number, default 50, max 200)
- Call `svc.listMessages(threadId, { afterMessageId, limit })`
- Return `{ messages, nextCursor }` — nextCursor is last message id if results.length === limit, else null

**POST /chat/threads/:threadId/messages** (API-04, API-05, API-06, API-07):
- `validate(sendMessageSchema)` middleware
- Look up thread, check exists, assertCompanyAccess on thread.companyId
- Get actor info, determine senderType: "agent" if actor is agent, "user" otherwise
- Set processingStatus: "enqueued" for user messages, "processed" for agent messages
- Insert message via `svc.createMessage()`
- Publish live event: `publishLiveEvent({ companyId: thread.companyId, type: "chat.message.created", payload: { threadId, messageId: message.id } })`
- If senderType === "user": fire-and-forget `void heartbeat.wakeup(thread.agentId, { source: "on_demand", triggerDetail: "system", reason: "chat_message", payload: { threadId, messageId: message.id }, requestedByActorType: actor.actorType, requestedByActorId: actor.actorId, contextSnapshot: { threadId, messageId: message.id, source: "chat.message" } })`
- CRITICAL: Do NOT fire wakeup when senderType === "agent" (prevents infinite loop)
- Return 201 with message object
