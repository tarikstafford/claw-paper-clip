  <action>
Create `packages/db/src/schema/chat_threads.ts` following the exact pattern from the research (Architecture Pattern 1). The table must have:
- `id`: uuid PK with defaultRandom
- `companyId`: uuid FK to companies.id, notNull
- `agentId`: uuid FK to agents.id, notNull
- `creatorAgentId`: uuid FK to agents.id, nullable (Pattern 2: dual creator)
- `creatorUserId`: text, nullable (Pattern 2: dual creator)
- `title`: text, nullable
- `createdAt`: timestamp with `{ withTimezone: true }`, notNull, defaultNow
- `updatedAt`: timestamp with `{ withTimezone: true }`, notNull, defaultNow
- Indexes: `chat_threads_company_agent_idx` on (companyId, agentId), `chat_threads_company_created_at_idx` on (companyId, createdAt)

Create `packages/db/src/schema/chat_messages.ts` following the research's recommended column set exactly. The table must have:
- `id`: uuid PK with defaultRandom
- `threadId`: uuid FK to chatThreads.id, notNull. Use `onDelete: "cascade"` per research anti-pattern guidance (deleting thread deletes messages)
- `senderType`: text, notNull (values: "user", "agent", "system") -- NO pgEnum (Pattern 3)
- `senderAgentId`: uuid, nullable (populated when senderType = "agent")
- `senderUserId`: text, nullable (populated when senderType = "user")
- `body`: text, notNull
- `tokenCount`: integer, nullable (Phase 3 fills this in)
- `processingStatus`: text, notNull, default "enqueued" (Pattern 3)
- `telegramUpdateId`: bigint with `{ mode: "number" }`, nullable (Pitfall 5)
- `createdAt`: timestamp with `{ withTimezone: true }`, notNull, defaultNow
- `updatedAt`: timestamp with `{ withTimezone: true }`, notNull, defaultNow
- Indexes: `chat_messages_thread_created_at_idx` on (threadId, createdAt), `chat_messages_processing_status_idx` on (processingStatus), `chat_messages_telegram_update_id_idx` as uniqueIndex on (telegramUpdateId) (Pattern 4 -- PostgreSQL UNIQUE indexes skip NULLs)

Import conventions: use `.js` extension in import paths (e.g., `import { chatThreads } from "./chat_threads.js"`). Only chat_messages imports chat_threads, never the reverse (Pitfall 4).

Add two export lines to `packages/db/src/schema/index.ts` at the end of the file:
```typescript
export { chatThreads } from "./chat_threads.js";
export { chatMessages } from "./chat_messages.js";
```
