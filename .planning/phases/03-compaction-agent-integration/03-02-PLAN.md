<task type="auto">
  <name>Task 2: Wire paperclipChatThreadContext into Claude adapter prompt assembly</name>
  <files>packages/adapters/claude-local/src/server/execute.ts</files>
  <action>
The Claude adapter in `packages/adapters/claude-local/src/server/execute.ts` currently only reads `paperclipSessionHandoffMarkdown` from the context object and joins it into the prompt (line 391). The new `paperclipChatThreadContext` key set by the chat route in Task 1 is passed through `contextSnapshot` into `context` but is never extracted or injected into the prompt. Without this step, COMP-05 is silently broken -- the context is set but never delivered to the agent.

**Follow the exact same pattern as `paperclipSessionHandoffMarkdown`:**

1. After line 391 (`const sessionHandoffNote = ...`), add:
```typescript
const chatThreadContext = asString(context.paperclipChatThreadContext, "").trim();
```

2. Update the `joinPromptSections` call (currently lines 392-396) to include the new section:
```typescript
const prompt = joinPromptSections([
  renderedBootstrapPrompt,
  sessionHandoffNote,
  chatThreadContext,    // NEW -- injects compacted chat thread into agent prompt
  renderedPrompt,
]);
```

Place `chatThreadContext` after `sessionHandoffNote` and before `renderedPrompt` so the agent sees the conversation context before the heartbeat-specific instructions.

3. Update the `promptMetrics` object to track the new section:
```typescript
const promptMetrics = {
  promptChars: prompt.length,
  bootstrapPromptChars: renderedBootstrapPrompt.length,
  sessionHandoffChars: sessionHandoffNote.length,
  chatThreadContextChars: chatThreadContext.length,   // NEW
  heartbeatPromptChars: renderedPrompt.length,
};
```

This is a minimal, safe change: `asString` returns `""` when the key is absent, `.trim()` handles whitespace, and `joinPromptSections` filters out empty strings. So existing runs without chat context are completely unaffected.

NOTE: All the helper functions (`asString`, `joinPromptSections`) are already imported. No new imports needed.
